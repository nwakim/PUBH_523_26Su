Data transformations on rows

  • We defined dplyr functions as operations that use columns as arguments
    • This is still true for row transformations
    • We use columns to define the rows we want to keep, remove, or modify
  • There are many functions that can be used to transform rows
  • We will cover the most common ones here:
    • filter()
    • arrange()
    • distinct()

filter()

filter(): quantitative data

  • filter() allows us to keep rows (aka subset our data) based on conditions we specify (in the columns)

  • We input our data frame as the first argument, and then we specify the conditions for filtering in the second argument

  • We can use the following comparison operators to specify our conditions:

    • >: greater than
    • <: less than
    • >=: greater than or equal to
    • <=: less than or equal to
    • ==: equal to
    • !=: not equal to

Example: filtering age

  • Let’s filter on a numeric variable, age: all rows with age values greater than 99

WAIT! Did those two operations have different rows?

  • It’s always good to check your output!

 

  • I did not notice that filter_out() and filter() with != had different outputs until I checked the number of rows in each output

 

  • filter_out() keeps the NA values, while filter() with != does not keep the NA values (because NA is not equal to anything, including itself)

When using !=, I suggest using filter_out()

arrange()

Sorting datasets using arrange()

  • arrange() changes the order of the rows in a dataset based on the values in one or more columns

 

  • You can arrange by ascending or descending order

 

  • Why use arrange()?
    • Visual clarity when you look at data (see a variable ordered)
    • Helpful in conjunction with other functions (like count())

distinct()

distinct() keeps unique rows

  • distinct() allows us to keep only unique rows based on the values in one or more columns
  • distinct() can run without any arguments: it will return all unique rows in the dataset
    • This can be helpful if you have repeat observations in your dataset and you want to keep only one of each
  • distinct() can also take specific columns as arguments: it will return unique rows based on the values in those columns

Wrap-up

  • We learned three functions that allow us to transform rows in our dataset:
    • filter() (and filter_out())
      • Keep rows based on conditions we specify in the columns
    • arrange()
      • Sort rows based on the values in one or more columns
    • distinct()
      • Keep only unique rows based on the values in one or more columns
