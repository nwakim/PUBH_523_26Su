Categorical data (factors)

  • Intro
  • Make degree factor
  • Change order of levels
  • Be careful!
  • Replace degree with factor version
  • Remove extra variables
  • One data type that we haven’t yet looked at are factor variables
    • factors are how R represents levels of categorical data.
  • For the most part, you can use character and factors interchangeably for categorical data.
  • However, there are two main differences:
    • factors define the permissible values in a vector with an argument called levels.
    • The levels define the order in which the permissible values are displayed, such as in a table or figure.
    • The default order of levels (values) is in alphanumeric order.
  • Below we use the factor() function within a mutate() to transform degree from a character variable into a factor variable.

Warning

Spelling matters!

  • If we try to use a factor level that doesn’t exist (“10” vs “10th”),
    • we won’t have any data in that category, and
    • the non-specified levels (“10”) get set to NA (missing).

Make variables with case_when()

case_when()

  • Intro
  • Code
  • Example 1
  • Example 2
  • Example 3
  • How can we dichotomize a continuous variable, such as income?
  • How can we combine categories of a variable, such as race?
  • How can we convert categories to numbers, such as female?

These are all examples where case_when() can be used.

  • The code follows a basic pattern for each category inside case_when():
    • condition ~ category name
  • The left side of the ~ is where we specify how to define the category.
  • The right side of the ~ is where we specify the category name.
  • Separate each “case” with a comma.
hrs_new |> tabyl(female)
 female    n   percent
 Female 1612 0.5909091
   Male 1116 0.4090909
  • Convert female to a new numeric variable called female whose values are
    • 1 when female is Female, and
    • 0 when female is Male

left side defines the condition ~ right side names the category

hrs_new <- hrs_new |>
  mutate(
    female = case_when(
      # when female is Female, make female equal to 1 
      female == "Female" ~ 1, # do not forget this comma!
      
      # when female is Male, make female equal to 0
      female == "Male" ~ 0
      )
    ) 

# checks
hrs_new |> tabyl(female) # look at the categories
 female    n   percent
      0 1116 0.4090909
      1 1612 0.5909091
class(hrs_new$female)
[1] "numeric"
# compare female and female variables
# hrs_new |> tabyl(female, female) |> adorn_title()

  • Dichotomize the numeric variable income into two categories with 30 as the cutoff.

left side defines the condition ~ right side names the category

hrs_new <- hrs_new |>
  mutate(
    income30 = case_when(
      income < 30 ~ "<30", # don't forget a comma between cases
      income >= 30 ~ "30+"
      ) 
  )

  • Recode income to have 4 levels:
    • <20 (not including exactly 20)
    • 20-30 (not including exactly 30)
    • 30+
    • unknown if missing
  • Code comments
    • Notice that for the middle category we use an & statement similar to how we used logic in filter
    • New: used .default option to specify what to assign in all other unspecified cases.
hrs_new <- hrs_new |>
  mutate(
    income3cat = case_when(
      income < 20 ~ "<20",
      (income >=20) & (income < 30) ~ "20-30",
      income >= 30 ~ "30+",
      .default = "unknown"  # apply this in all other cases
      ),
    income3cat = factor(income3cat) # make it factor after creating categories
    )
  
# check

Replace missing values with replace_na()

replace_na()

  • Sometimes we want to fill in missing values with a certain value.
  • We can use the replace_na() function inside of mutate() to specify this.
  • Example 1
  • Example 2
  • Example 3
  • For example, suppose if the income variable is missing,
    • we want to replace it with the average income across the entire data set.
hrs_new <- hrs_new |>
  mutate(
    income_filled = replace_na(
      income,
      mean(income, na.rm = TRUE))
    )           

# check
hrs_new |> 
  select(contains("income")) |> 
  tail()
