1. Typo errors

Missing comma

Error: unexpected symbol in:
"  geom_histogram(stat = "identity")+
  tidyquant::geom_ma(n=7, size = 2, color = "red" lty"

If you see “unexpected symbol”, the first thing to check is a missing comma.

R is case-sensitive — Mean() and mean() are different things!

Mean(c(1, 2, 3))
1
Capital M — R can’t find this function 
Error in `Mean()`:
! could not find function "Mean"

Mismatched parentheses

mean(c(1, 2, 3)
1
Missing closing ) — the console shows + waiting for more input 
Error in parse(text = input): <text>:2:0: unexpected end of input
1: mean(c(1, 2, 3)
   ^

Quick fixes:

  • Use Tab to autocomplete function names and avoid typos
  • Check that every ( has a closing )
  • Use RStudio’s bracket highlighting — click next to a ( and its match will highlight
  • Press Escape if the console gets stuck on +

2. Package errors (1/2)

Function not found

could not find function "x"...

This usually means you typed the function name incorrectly, or forgot to install or load the package.

clean_names(iris)
1
clean_names() is from janitor — if it’s not loaded, R can’t find it 
Error in `clean_names()`:
! could not find function "clean_names"

✅ Fix:

library(janitor)
clean_names(iris) |> names()
1
Load the package first, then call the function 
[1] "sepal_length" "sepal_width"  "petal_length" "petal_width"  "species"

Install once, load every session

  • install.packages("janitor") — run once, downloads to your computer
  • library(janitor) — run every session, makes it available in R

Think of it like: install = buy the book; library = open it.

2. Package errors (2/2)

Masked functions — when two packages have the same function name

Error in select(data, var) : unused argument (var)

This is a classic error! Both dplyr and MASS have a select() function. Whichever package was loaded last wins — and if MASS was loaded after dplyr, you’ll get the wrong select().

library(MASS)
library(dplyr)
iris |> select(Species)
1
Which select() is this? Depends on load order!

✅ Fix — be explicit with package::function():

iris |> dplyr::select(Species) |> head(3)
1
No ambiguity — always uses dplyr’s version 
  Species
1  setosa
2  setosa
3  setosa

Other common masking conflicts:

Package A Package B Function
dplyr MASS select()
dplyr plyr summarise()
dplyr stats filter()

The conflicted package forces R to throw an error whenever a masked function is used — great for catching these silently wrong results.

3. Object errors

Object not found

object 'x' not found

This means R can’t find a variable you referenced. Common causes:

my_summary
1
This object was never created — code above may not have run 
Error:
! object 'my_summary' not found
install.packages(dplyr)
1
Missing quotes — R looks for an object named dplyr 
Error:
! object 'dplyr' not found

✅ Fix: install.packages("dplyr")

File not found

No such file or directory
read_csv("my data.csv")
1
Typo, wrong path, or file doesn’t exist yet

Check:

  • Spelling of the file name
  • Forward slashes / not backward \
  • Correct file extension (.csv, .xlsx, .rds)
  • That you are in the right R Project

Subscript out of bounds

Error in 'x': subscript out of bounds

You tried to access an element that doesn’t exist — e.g., asking for the 10th element of a 5-element vector.

4. Function syntax errors

Wrong or missing argument names

A very common mutate() mistake — forgetting to repeat the column name inside recode():

iris |>
  mutate(Species = recode("setosa" ~ "S"))
1
recode() needs the column name as its first argument 
Error in `mutate()`:
ℹ In argument: `Species = recode("setosa" ~ "S")`.
Caused by error in `UseMethod()`:
! no applicable method for 'recode' applied to an object of class "formula"

✅ Fix:

iris |>
  mutate(Species = recode(Species,
                          "setosa" = "S")) |>
  count(Species)
1
Pass Species as the first argument, then OLD = NEW pairs 
     Species  n
1          S 50
2 versicolor 50
3  virginica 50

Wrong argument name

mean(c(1, 2, NA), remove.na = TRUE)
1
remove.na is not a real argument — R ignores it and NA propagates 
[1] NA

✅ Fix:

mean(c(1, 2, NA), na.rm = TRUE)
1
The correct argument is na.rm 
[1] 1.5

How to find the right argument names

Run ?function_name and read the Arguments section — it lists every accepted argument and what it expects.

5. Logic errors

These don’t always produce an error — the code runs, but gives you the wrong answer!

if statement errors

Error in if

This usually means an if statement was applied to something that wasn’t TRUE or FALSE:

x <- c(1, 2, 3)
if (x > 1) print("big")
1
x > 1 returns a vector — if expects a single TRUE/FALSE 
Error in `if (x > 1) ...`:
! the condition has length > 1

✅ Fix — use any() or all():

if (any(x > 1)) print("at least one is big")
1
any() collapses the vector to a single TRUE/FALSE 
[1] "at least one is big"

Silent logic errors (no error message!)

ages <- c(23, 31, 40, 45, 28)
mean(ages[ages = 30])
1
= should be == — this silently does something wrong! 
[1] NA

✅ Fix:

mean(ages[ages == 30])
1
== is “equal to”; = is assignment 
[1] NaN

Logic errors are the sneakiest bugs. Always check intermediate results with print(), head(), or View() to confirm your data looks right at each step.

6. Factor errors

Factors look like character strings, but they have fixed levels — adding new values creates NA!

Adding a value not in the levels:

age_cat <- factor(c("child", "adult", "elder"))
age_cat[1] <- "infant"
age_cat
1
“infant” is not a defined level — NA is introduced 
[1] <NA>  adult elder
Levels: adult child elder

This error message gives it away:

invalid factor level, NA generated

If you see it, you tried to add a value that wasn’t in the factor’s pre-defined levels.

✅ Fix — convert to character first, then back to factor:

age_cat <- factor(c("child", "adult", "elder"))
age_cat <- as.character(age_cat)
age_cat[1] <- "infant"
age_cat <- factor(age_cat)
levels(age_cat)
1
Convert to character so any value is accepted
2
Re-factor with all levels present 
[1] "adult"  "elder"  "infant"

Unexpected alphabetical ordering:

survey <- data.frame(
  rating = factor(c("Agree", "Neutral", "Disagree")))
levels(survey$rating)
1
Default is alphabetical — probably not what you want! 
[1] "Agree"    "Disagree" "Neutral"

✅ Fix with levels =:

survey$rating <- factor(survey$rating,
  levels = c("Disagree", "Neutral", "Agree"))
levels(survey$rating)
[1] "Disagree" "Neutral"  "Agree"

7. Plotting errors

Not enough colors for factor levels

Error: Insufficient values in manual scale. 
3 needed but only 2 provided.
ggplot(iris, aes(x = Sepal.Length, fill = Species)) +
  geom_histogram() +
  scale_fill_manual(values = c("orange", "purple"))
1
iris has 3 species, but only 2 colors were provided 
Error in `palette()`:
! Insufficient values in manual scale. 3 needed but only 2 provided.

✅ Fix — provide one color per level (check if NA became a factor level too!):

scale_fill_manual(values = c("orange", "purple", "green"))

Extra + at the end of a ggplot chain

Can't add x object
ggplot(iris, aes(x = Sepal.Length)) +
  geom_histogram() +
1
Trailing + with nothing after it 
Error in parse(text = input): <text>:3:0: unexpected end of input
1: ggplot(iris, aes(x = Sepal.Length)) +
2:   geom_histogram() +
  ^

✅ Fix — delete the final +

Extra %>% or |> in a dplyr chain

Similarly, re-arranging piped steps and leaving a stray pipe at the end (or middle) will break your code:

iris |>
  filter(Sepal.Length > 5) |>
1
Dangling pipe — R waits for the next step but there isn’t one 
Error in parse(text = input): <text>:3:0: unexpected end of input
1: iris |>
2:   filter(Sepal.Length > 5) |>
  ^

8. R Markdown / Quarto errors

Wrong chunk option names

Error in options[[sprintf("fig.%s", i)]]

Check that your chunk options use the correct names:

❌ Wrong ✅ Correct
fig.width = 5 out.width = "5in"
fig.height = 3 out.height = "3in"

Duplicate chunk labels

Error: Duplicate chunk label 'my-plot'

Every code chunk label in your document must be unique. Labels also cannot contain spaces.

```{r}
#| label: my-plot-1
```
1
Add a number or descriptor to make each label unique

YAML indentation errors

---
format: html
  toc: true   # ❌ wrong indentation
---

✅ Fix:

---
format: 
  html:
    toc: true   # ✅ correct nesting
---

Chunk runs in console but fails on render

This usually means your document is relying on objects created earlier in your R session that don’t exist when rendering. The render starts with a fresh environment.

Always use Session → Restart R → Render to catch these before submitting.

Wrap-up: A troubleshooting checklist

When your code gives an error, work through this list:

Step What to do
Read Read the error message — what function? What keyword?
Typos Check spelling, capitalization, commas, and matching parentheses
Packages Is the right package installed and loaded? Any masking conflicts?
Objects Does the variable exist? Run ls() to list everything in your environment
Arguments Run ?function_name and check argument names and types
Restart Session → Restart R → rerun from top
Google Search the exact error message + “in R”
Ask Make a reprex and ask a classmate, in office hours, or on Posit Community

Even experienced programmers encounter errors constantly. Getting faster at reading error messages is one of the most valuable skills you will build in this course.

Typo errors: Incomplete commands

  • When the console is waiting for a new command, the prompt line begins with >
    • If the console prompt is +, then a previous command is incomplete
    • You can finish typing the command in the console window

Example:

> 3 + (2*6
Error in parse(text = input): <text>:2:0: unexpected end of input
1: 3 + (2*6
   ^
> 3 + (2*6
+ )
[1] 15

Error Type 1: Typo Errors

The most common error — and the easiest to fix!

Misspelled function or variable name:

# Typo in function name
my_data <- data.frame(x = 1:5, y = 6:10)
sumamry(my_data)
Error in `sumamry()`:
! could not find function "sumamry"
# Typo in variable name
my_datta |> head()
Error:
! object 'my_datta' not found

Mismatched parentheses or brackets:

mean(c(1, 2, 3)
Error in parse(text = input): <text>:2:0: unexpected end of input
1: mean(c(1, 2, 3)
   ^

Quick Fixes

  • R is case-sensitive: Mean() ≠ mean()
  • Use RStudio’s auto-complete (Tab key) to avoid typos
  • Check that every ( has a matching )

Object is not found

  • This happens when text is entered for a non-existent variable (object)

Example:

hello
Error:
! object 'hello' not found
  • Can be due to missing quotes
install.packages(dplyr) 
Error:
! object 'dplyr' not found
# correct code is: install.packages("dplyr")
